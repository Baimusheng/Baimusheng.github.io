一、基础搭建

1. 安装必要工具并搭建本地博客

  • git

  • node.js

  • hexo

    • 选择一个文件夹，把相关的博客信息存储到这个文件夹下,本人路径：/Users/clhao/Blog

  • 如上图所示，hexo本地博客就已经搭建完成，十分的简单～

2. 将本地的博客部署到Github上

  • 在github上创建项目代码库

  • 需要配置ssh密钥，配置完成之后才能将文件上传的github上(这一步比较关键)

  • 获取github项目代码库的ssh路径

  • 在.config.yml文件的最后更改以下代码

  • 最后，把本地的文件部署到github上

3. 文章上传

  • 文章的上传这个问题困扰了我挺久的，因为你不可能就只简单的做文字笔记，笔记里面肯定还有图片，如何把图片同步上传上去这是一个问题，但现在这个问题解决了。但是用的是Typora(之前用的是obsidian)，以后可能用云图床来代替。

a) 在你本地的博客目录下，创造一个文件夹，用来存放文章的图片

b) 对全局图像进行设置（目的是每次放到Typora中的文件都同步复制到相应的路径下，就不用手动复制了，这里建议绝对路径)

c) 这篇文章的图片根目录，设置到你存放图片的目录下，注意要在source目录下

4. 域名设置

【1】首先在阿里云上买一个域名(或者腾讯云，还能是一些其他的网站，那些比较便宜)

【2】在配置好自己域名的DNS解析

【3】再在source目录下创建一个CNAME文件，其中写入自己的域名

【4】在github上写入自己的域名，然后就可以通过域名来访问自己的博客了

  • 以上就是博客基础的步骤了，当然还是需要美化的～

二、博客美化

  • 博客美化主要是用了Hexo官方的butterfly主题，可能会加点魔改(https://butterfly.js.org),但主要是满足日常记录文章的需要

1.安装butterfly主题

  • 有多种安装方式，这里介绍git安装

    • 把butterfly主题clone下来

    • 修改根目录下的.config.yml文件中的theme

    • 需要安装pub和stylus的渲染器

  • 把theme中的_config.yml文件中的内容复制出来放到根目录下，名字为_config.butterfly.yml。hexo会自动合并主题中的_config.yml和_config.butterfly.yml文件里面的配置，且_config.butterfly.yml优先级较高,这样butterfly已经安装完成了～
